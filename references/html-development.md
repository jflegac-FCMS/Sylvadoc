# Préférences pour le développement HTML

Préférences pour confectionner des documents HTML modernes, accessibles et performants.

## Principes généraux

| Aspect               | Choix                                                                                       |
|----------------------|---------------------------------------------------------------------------------------------|
| Syntaxe HTML         | HTML5 standard                                                                              |
| Structure            | Utilisation sémantique des balises HTML                                                     |
| Accessibilité        | Respect des normes d'accessibilité (WCAG)                                                   |
| Performance          | Optimisation des ressources et du chargement                                                |
| Outils               | Validation HTML, linters et outils d'audit                                                  |
| Méthodologie         | BEM (Block Element Modifier) pour la structuration des classes CSS                          |
| Compatibilité        | Tests multi-navigateurs et responsive design                                                |
| SEO                  | Optimisation pour les moteurs de recherche                                                  |
| Internationalisation | Support des langues et encodage UTF-8                                                       |
| Tests                | Validation HTML et audits d'accessibilité                                                   |
| Réutilisabilité      | Création de composants HTML réutilisables                                                   |
| Maintenance          | Structuration claire pour faciliter la maintenance                                          |
| Sécurité             | Protection contre les vulnérabilités courantes (XSS, injections, etc.)                      |
| Performance mobile   | Optimisation pour les appareils mobiles                                                     |
| Progressive Enhancement | Support des fonctionnalités avancées tout en assurant la compatibilité de base              |
| Frameworks et bibliothèques | N'utiliser aucune librairie de composants ou de bibliothèques de type Tailwind ou Bootstrap |
|----------------------|---------------------------------------------------------------------------------------------|
